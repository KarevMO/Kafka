#Данный пайплайн реализует устойчивое решение миграции данных с защитой от дубликатов.

В PostgreSQL хранится таблица user_logins, содержащая события пользователей.
Эти данные необходимо передавать в другую систему (ClickHouse), но без повторной отправки.
Для этого используется логическое поле sent_to_kafka, которое гарантирует уникальную передачу.

Пайплайн состоит из двух компонентов:
Producer — читает новые записи из PostgreSQL и отправляет их в Kafka.
Consumer — читает сообщения из Kafka и загружает их в ClickHouse.

Для запуска пайплайна нужно: 
1.В исходной таблице PostgreSQL создать новое поле (столбец) командой: 
'ALTER TABLE user_logins
ADD COLUMN sent_to_kafka BOOLEAN DEFAULT FALSE;'
2. Затем запустить файл prodecer.py
3. Далее запустить файл consumer.
